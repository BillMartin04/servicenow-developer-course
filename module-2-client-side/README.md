# Module 2 · Client-Side Scripting

**Estimated time:** ~30 min  ·  **Lessons:** 4

Control form behaviour in the browser using Client Scripts and UI Policies, including the catalog-specific variants used in the Service Catalog.

## Learning objectives

By the end of this module, you will be able to:

- Create onLoad, onChange, onSubmit, and onCellEdit client scripts
- Use UI Policies to control field visibility, mandatory, and read-only states
- Apply catalog client scripts and catalog UI policies to catalog items
- Decide between a client script and a UI Policy for a given requirement

## Prerequisites

Modules 0–1. You should understand client-side execution and basic g_form usage.

## Knowledge check

Before moving on, make sure you can answer these:

- When would you choose a UI Policy over a client script?
- What are the four client script types and when does each fire?
- How do catalog client scripts differ from standard client scripts?
