# How to use this course

This page explains exactly how to work through the material so you get maximum value with minimum wasted effort.

## What you'll need

| Requirement | Why | Where |
| --- | --- | --- |
| A free PDI | Every exercise runs in your own ServiceNow instance | [Module 0 lesson](module-0-getting-started/04-set-up-your-pdi.md) |
| ~5–6 hrs + capstone time | To complete all modules and the project | See the [Syllabus](syllabus.md) |
| A notebook or doc | Capture answers to knowledge checks and your capstone design | Any tool you like |

## The learning loop

Each lesson is designed around a simple, repeatable loop:

1. **Read the objectives** — know what you're about to be able to do.
2. **Watch / read** — the video or written guide for the topic.
3. **Study the key concepts** — the ideas you must internalise.
4. **Do the hands-on** — build it in your PDI. This is where the learning sticks.
5. **Check yourself** — at the end of each module, answer the knowledge-check questions from memory.

{% hint style="info" %}
Do the hands-on exercises. Watching alone will not make you a developer — building will.
{% endhint %}

## How each module is organised

Every module page gives you, in order:

- **Estimated time and lesson count**
- **Learning objectives** — "By the end of this module, you will be able to..."
- **Prerequisites** — what to finish first
- **Lesson table** — each lesson, its type, time, and what you'll learn
- **Knowledge check** — questions to confirm you've got it
- **Module recap** — a one-line summary of what you achieved
- **Next up** — a direct link to the following module

## Recommended path

Work **in order**, Module 0 → Module 7. Each module assumes the previous ones. If you're experienced, use the [Syllabus](syllabus.md) table to skip to the modules you need — but at least skim earlier knowledge checks to confirm no gaps.
